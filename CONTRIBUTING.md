# Contributing to the Price Server User Guide

Thanks for your interest in contributing! This repo is documentation for the [Grid Coordination](https://grid-coordination.energy) price server — getting-started guides, language tutorials, and reference material for users connecting to the OpenADR 3 VTN.

## How to contribute

### Discussions

Use [Discussions](https://github.com/grid-coordination/price-server-user-guide/discussions) for:

- Questions about how to use the price server
- Feature requests and suggestions
- Sharing what you're building or how you're using the data
- Asking other users for advice

Discussions are open-ended — a good place to think out loud or scope something before it becomes a concrete change. Aligned outcomes from a Discussion often turn into one or more Issues.

### Issues

Use [Issues](https://github.com/grid-coordination/price-server-user-guide/issues) for actionable changes:

- Documentation errors, omissions, or unclear explanations
- Bugs or unexpected behavior in tutorial example code
- Discussion outcomes that have alignment and a clear scope

If you're not sure whether something is an Issue or a Discussion, start with a Discussion — we can convert it later.

### Pull requests

Pull requests are welcome.

- For small fixes (typos, broken links, dead code, version bumps), open a PR directly.
- For substantive changes (new sections, restructuring, new tutorials), open a Discussion or Issue first so we can align on scope before you invest the effort.
- Match the existing tone and structure. Tutorials in this repo are short, runnable, and grounded in the live server — please test your examples against `https://price.grid-coordination.energy/openadr3/3.1.0`.
- One commit per logical change is fine; we don't require squash or any particular branch naming.

## Code of conduct

Be respectful and constructive. We're a small project and appreciate everyone who takes the time to file an issue or send a PR.

## Important notice

The price server and this documentation are provided on an "as-is" basis. Updates and maintenance, including responses to issues filed on GitHub, will take place on an "as time and resources permit" basis.
